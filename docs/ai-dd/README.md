# AI-DD (AI-Driven Development)

Documentation technique destinée aux agents IA pour comprendre et modifier le projet.

## Structure

```
AI-DD/
├── README.md           # Ce fichier
└── trpc/
    ├── ARCHITECTURE.md # Architecture tRPC du dashboard
    └── CONVENTIONS.md  # Conventions et bonnes pratiques
```

## Usage

Ces documents servent de référence pour :
- Comprendre l'architecture du projet
- Suivre les conventions établies
- Éviter les anti-patterns
- Ajouter de nouvelles fonctionnalités correctement

## Mise à jour

Mettre à jour ces documents quand :
- L'architecture change significativement
- De nouvelles conventions sont établies
- Des patterns problématiques sont identifiés
